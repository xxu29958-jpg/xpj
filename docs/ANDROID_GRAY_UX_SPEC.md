# Android 灰度版 UX/UI 规格

日期：2026-05-05

## 1. 总目标

Android 灰度版要从“联调工具”改成“普通人能直接使用的私人生活账本 App”。

普通用户不需要知道服务器怎么运行，也不需要看到接口名、token、域名、Cloudflare、端口或日志。

底部导航保留：

```text
待确认
账本
统计
设置
```

上传入口放在“待确认”页主操作，也可以在有数据时作为轻量按钮保留。

## 1.1 外部产品参考边界

允许参考外部成熟产品的体验模式，但不照搬视觉、素材、商标或文案。

参考方向：

- 记账类 App 的首页通常优先展示本月概况、最近账单和明确主操作。
- 预算类 App 倾向把复杂同步和账户状态收敛成“已连接”“最近同步”这类用户语言。
- 分账类 App 的列表更轻量，强调人物能理解的状态和下一步动作。
- 大厂设计规范通常把技术细节放在高级入口，普通设置页只展示用户能判断和操作的状态。

小票夹自己的落地原则：

- 主叙事是“截图等待确认”，不是“接口返回 pending”。
- 主状态是“账本可用”，不是“服务器诊断通过”。
- 主动作是“上传截图”和“确认入账”，不是“调用 API”。
- 技术诊断属于服务拥有者，不属于普通灰度用户。

## 2. 待确认页

目标：让用户知道“截图上传后在这里确认”，不要像接口列表。

要求：

- 默认紧凑模式。
- 顶部标题使用“待确认账单”。
- 增加主按钮“上传截图”。
- Android 使用系统 Photo Picker 选择图片。
- iOS 快捷指令说明作为次要入口，不占主视觉。
- 账单卡片显示缩略图、金额草稿、商家草稿、分类、时间、状态。
- 缩略图小尺寸展示，点击进入详情页再看大图。
- OCR 未识别金额时显示“等待你确认金额”。
- OCR 低置信度时显示“请核对”。
- 空状态文案：“截图上传后，会出现在这里等你确认。”
- 不显示接口名、id、token、URL。

验收：

- 无 pending 时不是空白页。
- 有 pending 时一眼能看到要处理的账单。
- 上传成功后自动刷新列表。
- 上传失败显示生活化中文错误。

## 3. 编辑确认页

目标：确认流程像记账，不像编辑数据库字段。

要求：

- 图片预览清晰。
- 金额作为第一输入项。
- 商家、分类、消费时间、备注依次展示。
- 分类用用户能理解的标签。
- OCR 填出来的内容标记为“识别建议”。
- 用户修改后再确认入账。
- 确认按钮文案为“确认入账”。
- “忽略这张”入口弱化，避免误点。
- 可能重复提示：“这张可能已经记过了”。
- OCR 原文折叠到“查看识别原文”。

验收：

- 未填金额不能确认。
- OCR 结果不会自动入账。
- 用户明确知道确认后才进入账本。

## 4. 账本页

目标：像日常账本，不像分页查询器。

要求：

- 月份显示为“2026年5月 ▼”。
- 点击月份后用底部弹窗选择月份。
- 分类筛选用横向标签或二级筛选。
- 不要输入框和标签重复出现。
- 账单列表按日期分组，或至少按时间清晰排序。
- 每条账单展示金额、商家、分类、时间、备注摘要。
- 当前无数据时说明具体原因，例如“本月暂无餐饮账单”。
- 导出按钮在无数据时禁用，不要点了才失败。
- 不显示 `page`、`page_size`、`month`、`category` 这些接口概念。

验收：

- 用户不需要理解接口参数也能筛选账单。
- 无数据状态能说明为什么没数据。
- 本地缓存可在网络失败时兜底。

## 5. 统计页

目标：像生活总结，不像后台报表。

要求：

- 顶部总支出卡片。
- 显示账单数量。
- 显示分类占比。
- 显示最近 7 天支出。
- 显示最大一笔。
- 显示高频商家。
- 0 元分类隐藏或弱化。
- 文案生活化，例如“这个月主要花在餐饮上”。
- 不暴露接口名、字段名。

验收：

- 一屏能看懂本月花费概况。
- 没有数据时显示生活化空状态。

## 6. 设置页

目标：普通用户只看到自己的账本状态，不看到服务器控制台。

普通灰度用户默认只展示：

- 当前账本
- 小票夹已连接
- 最近上传
- 最近同步
- 存储状态：正常
- 外观
- 数据与安全
- 清除本地数据
- 清除绑定 / 退出
- 关于

必须移出主展示：

- 服务器域名
- Cloudflare
- Tunnel
- 数据库大小
- pending 接口
- confirmed 接口
- token
- 本机服务
- 端口
- 后端日志
- 诊断脚本

要求：

- 清除本地数据必须二次确认。
- 清除绑定必须二次确认。
- 设置页文案要像 App 设置，不像 API 联调面板。
- 技术细节只允许在 debug build、隐藏高级入口、Windows 诊断脚本、运维文档或后续 Web 管理页中出现。

验收：

- 灰度用户看不到服务器域名和 token。
- 灰度用户看不到 Cloudflare、端口、日志、诊断脚本。
- 设置页只表达“我的账本是否正常可用”。

## 7. 外观和主题

目标：港湾主题作为默认审美，形成耐看的私人账本气质。

要求：

- 港湾主题作为默认推荐。
- 主题卡片不只显示色块，要显示小型 App 预览。
- 支持背景选择。
- 后续支持用户自定义背景图。
- 背景必须有遮罩和对比度控制，不能影响文字可读性。
- 卡片、按钮、金额、分类标签统一视觉语言。
- 各页面主题切换后都要保持可读。

主题预览卡片应展示：

- 小型账单卡片。
- 主按钮。
- 分类标签。
- 背景和表面色关系。
- 当前主题选中态。

## 8. 文案替换表

```text
pending -> 待确认
confirmed -> 已入账
rejected -> 已忽略
Upload-Token -> 上传凭证，仅文档和高级入口可见
APP_TOKEN -> 访问口令，仅绑定页和高级文档可见
server -> 小票夹服务
API -> 普通 UI 不展示
OCR -> 识别建议 / 自动识别
duplicate -> 可能重复
reject -> 忽略这张
confirm -> 确认入账
```

## 9. Release 与 Debug 差异

小票夹 Android 现在按受众拆成两个 flavor：

```text
gray      灰度用户版，包名 com.ticketbox
internal  内部运维/联调版，包名 com.ticketbox.internal
```

灰度用户版：

- 不显示开发诊断入口。
- 不打印网络调试日志。
- 不展示服务器域名。
- 不展示 token。
- 不展示接口名。

内部版：

- 可以显示内部工具和连接诊断。
- 可以展示绑定地址、诊断详情和耗时。
- 可以写 Logcat，但不得打印 token、请求头或请求体。

构建任务：

```powershell
cd android
.\gradlew.bat :app:assembleGrayDebug
.\gradlew.bat :app:assembleInternalDebug
```

## 10. 实现边界

- Screen 不直接调用 Retrofit。
- Screen 不直接调用 Room。
- ViewModel 只发事件和 UI state。
- Repository 负责上传、同步和错误转换。
- 上传图片读取使用 Android Photo Picker 返回的 URI。
- App 上传走 `Authorization: Bearer APP_TOKEN`。
- Android 不保存 `Upload-Token`。
