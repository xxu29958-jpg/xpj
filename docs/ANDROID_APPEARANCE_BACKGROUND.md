# Android 外观、背景与沉浸模式规格

日期：2026-05-06

本文是 Android 设置二级菜单、主题外观、自定义背景、预览、裁剪和沉浸模式的产品与工程验收文档。总体 UX 仍以 `docs/ANDROID_GRAY_UX_SPEC.md` 为准，工程边界仍以 `docs/ANDROID_RULES.md` 和 `docs/DECISIONS/0019-android-custom-background-local-only.md` 为准。

## 1. 目标

外观能力要让小票夹更像私人生活账本，而不是把设置页变成主题调试台。

- 设置主页面保持低密度，只展示“外观”摘要和入口。
- 外观进入二级菜单承载主题、背景、预览、裁剪和沉浸强度。
- 港湾主题仍是默认推荐。
- 自定义背景只属于本机 UI 个性化，不进入后端、Room、SQLite 或导出。
- 所有页面在主题和背景切换后都必须优先保证金额、正文、按钮和错误反馈可读。

## 2. 设置二级菜单结构

设置主页面只保留摘要：

```text
外观
港湾 / 平衡 / 自定义背景已开启
```

点击后进入二级菜单：

```text
外观
  主题
  背景
  预览
  沉浸强度
  动效与可读性
```

主设置页不得直接堆满色块、裁剪控件、透明度滑杆或调试参数。二级菜单可以使用分组列表、底部弹窗和预览面板，但普通用户不应看到 DataStore、文件路径、alpha、scrim、Uri 等工程词。

二级菜单返回后，设置主页面摘要必须同步更新：

- 当前主题名。
- 当前沉浸强度。
- 是否启用自定义背景。
- 夜幕主题或深色模式状态。

验收：

- 设置首屏仍能一眼看到当前账本、连接状态、最近同步、外观、数据与安全。
- 外观配置都能从“外观”二级菜单找到。
- 用户返回设置主页面后能看到外观摘要变化。
- 灰度版不显示背景文件路径、调试开关或渲染参数。

## 3. 主题包

主题包包含五套：

| 主题 | 定位 | 要求 |
| --- | --- | --- |
| 松雾 | 雾青暖白 | 安静、轻量，适合低干扰账本。 |
| 港湾 | 海蓝暖光 | 默认推荐，作为灰度版主审美。 |
| 柚光 | 暖纸橙黄 | 明亮日间可读，避免整页泛黄。 |
| 莓果 | 柔粉莓色 | 保留亲和感，不牺牲文字对比。 |
| 夜幕 | 深色松林月色 | 低光场景，状态栏和底部导航同步深色可读。 |

主题预览卡片必须展示：

- 小型待确认账单卡片。
- 主按钮。
- 分类标签。
- 金额文本。
- 背景和表面色关系。
- 当前主题选中态。

验收：

- 港湾是默认主题。
- 主题卡片不是单纯色块。
- 切换主题后待确认、账本、统计、编辑和设置页文字清晰。
- 夜幕主题下系统状态栏、导航栏、底部导航、SnackBar 和弹窗都可读。

## 4. 自定义背景

产品要求：

- 背景位于主题之下，是第三层氛围，不替代主题。
- 用户可选择“使用主题默认背景”“选择本机图片”“清除自定义背景”。
- 背景不能抢正文、金额、确认按钮、错误提示和输入框的注意力。
- 背景只影响 App Shell 氛围，不改变账单金额、主按钮、错误/成功/警告等语义色。

工程要求：

- 使用 Android Photo Picker 选择单张图片。
- 不申请全相册权限。
- Picker 返回的 `Uri` 只用于读取，不长期保存。
- 选中后复制或处理到 App 私有目录，例如 `context.filesDir/backgrounds/custom_background.jpg`。
- 背景配置使用 DataStore 保存来源、私有文件名、裁剪参数、沉浸强度、视差开关和减少动效开关。
- 背景不上传后端，不进入 Room，不进入 SQLite，不进入 CSV 导出。
- 清除背景时删除私有目录中的背景文件，并清空 DataStore 中对应配置。
- 背景图片按最大边 2160px 采样解码，避免相册原图直接进入 Compose 背景层。

验收：

- 选择图片后断网仍能显示背景，因为文件在本机私有目录。
- 清除背景后私有背景文件不再被使用。
- App 不请求 `READ_MEDIA_IMAGES` 这类全相册权限。
- 导出、上传、同步和后端接口不会包含背景文件。

## 5. 背景预览

预览目标：

用户应用背景前，能看到它在真实账本界面中的可读性，而不是只看一张图片。

预览内容至少覆盖：

- 待确认卡片。
- 账本列表行。
- 统计摘要卡。
- 编辑页表单片段。
- 设置页列表片段。

预览操作：

- 选择图片后先进入预览，不直接应用。
- 预览页或底部面板允许切换“待确认 / 账本 / 编辑 / 设置”代表画面。
- 用户可以“应用背景”“重新选择”“裁剪”“取消”。
- 取消时不改变当前已生效背景。

工程要求：

- 预览使用和正式 App Shell 相同的背景层、遮罩和页面角色规则。
- 不为预览单独写一套视觉算法。
- 预览中使用假账单内容，不读取真实账单隐私。
- 预览图不上传、不缓存到后端。

验收：

- 亮色、暗色、高对比和复杂纹理图片都能在预览中看出文字可读性。
- 用户取消预览后当前外观不变。
- 预览中的效果和应用后的实际页面一致。

## 6. 裁剪

裁剪目标：

让用户把本机图片调整到适合手机竖屏背景，同时不引入沉重图片编辑器。

产品要求：

- 提供移动、缩放、重置和确认。
- 默认使用居中裁剪，用户不裁剪也能应用。
- 裁剪页面应有账本内容安全区提示，避免主体图案压住金额和按钮。
- 裁剪只处理背景图，不修改原相册图片。

工程要求：

- 裁剪源只来自 Photo Picker 授权的本次图片或 App 私有目录中的背景副本。
- 最小实现可以只保存裁剪后的 `custom_background.jpg`。
- 如果后续支持重新裁剪，原始副本也只能保存在 App 私有目录，且必须被“清除自定义背景”一并删除。
- 裁剪输出仍按最大边 2160px 限制，并统一处理图片方向。
- 裁剪失败时保留原有背景，不产生半应用状态。

验收：

- 不裁剪直接应用时效果合理。
- 裁剪后预览和实际页面一致。
- 裁剪失败不会破坏当前外观。
- 清除背景会同时清理裁剪产物。

## 7. 沉浸模式

沉浸强度提供三档：

| 档位 | 文案 | 适用场景 |
| --- | --- | --- |
| 氛围 | 背景更明显 | 待确认、统计等浏览型页面。 |
| 平衡 | 默认推荐 | 兼顾好看和清晰，默认值。 |
| 专注 | 背景弱化 | 账本、编辑、设置、绑定、生物识别等录入和安全页面。 |

页面角色规则：

- 待确认和统计可以更沉浸。
- 账本页中等沉浸，账单列表仍要可扫读。
- 编辑、设置、绑定和生物识别必须更克制。
- 弹窗、底部弹窗、日期选择器、确认对话框必须使用实体表面，不能让背景干扰操作。

工程要求：

- 背景渲染只能放在统一 `ImmersiveBackgroundScaffold` / `TicketboxBackgroundLayer`。
- Screen 只声明页面角色或使用统一容器，不各自拼背景。
- 页面角色决定遮罩强度、卡片实体感和背景可见度。
- 减少动效开启时关闭视差、背景缩放动画和强过渡。
- 主题语义色保持稳定，不从用户背景随机派生金额、错误、成功或警告色。

验收：

- 三档切换后所有主页面都保持可读。
- 编辑页和设置页不会出现背景压住输入框、按钮或风险操作。
- 减少动效开启后没有视差和明显背景动效。
- 页面切换时背景不闪白、不拉伸变形。

## 8. 可读性与无障碍底线

必须满足：

- 正文、小字、输入框、金额、分类标签和错误提示在复杂背景上仍清晰。
- 普通正文对比度按 4.5:1 作为最低验收线，大号金额和图标按 3:1 作为最低验收线。
- 背景不能导致按钮边界消失。
- 选中态、禁用态、错误态、成功态和加载态都要有清楚差异。
- 文案不能溢出按钮、卡片或设置行。

人工验收至少使用三类图片：

- 高亮浅色背景。
- 暗色低对比背景。
- 复杂纹理或高饱和背景。

每类图片至少检查：

- 待确认页。
- 账本页。
- 编辑页。
- 设置页。
- 弹窗和底部弹窗。

## 9. 工程验收清单

必须通过：

- 背景选择使用 Photo Picker。
- 不申请全相册权限。
- 背景文件只在 App 私有目录。
- DataStore 中只保存小型配置，不保存图片二进制。
- Room schema 不因背景功能变化。
- 后端 API、上传、导出、同步都不包含背景。
- 统一背景层覆盖 App Shell。
- 页面角色控制沉浸程度。
- 大图采样后再进入 UI。
- 清除背景会删除私有文件并恢复主题默认背景。
- gray 版不显示调试参数；internal 版也不得打印背景路径中的用户隐私信息。

## 10. 产品验收清单

必须通过：

- 设置主页面仍简洁，外观能力在二级菜单内。
- 用户能选择主题。
- 用户能选择本机背景。
- 用户能预览背景在真实账本界面中的效果。
- 用户能裁剪、重选、取消和应用。
- 用户能切换氛围、平衡、专注。
- 用户能清除自定义背景。
- 港湾默认主题效果完整。
- 复杂背景不会影响确认入账、账本扫读、金额输入和风险操作。
