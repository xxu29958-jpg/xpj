# 小票夹 Android UX 改进计划

日期：2026-05-04

## 原则

小票夹第一版已经跑通上传、确认、统计、导出和同步闭环。后续 UI 改造优先处理可理解性和可预测性，再处理视觉丰富度。

## P0 已纳入执行

- 待确认页空状态必须告诉用户账单从哪里来，不只显示“没有数据”。
- 账本页导出必须先判断当前是否有可导出账单，不能让用户点击后才看到“操作失败”。
- 设置页面向用户时不直接暴露 `APP_TOKEN`、`pending`、`confirmed` 等内部术语。
- 连接诊断默认显示摘要，技术明细必须折叠。
- 清除本地缓存必须二次确认。

## P1 本轮推进

- 账本页合并月份选择器，去掉输入框和快捷标签重复表达。
- 分类筛选统一为横向标签或底部选择面板。
- 主题卡片增加真实账单卡片预览，而不是只显示色块。
- 港湾主题作为默认主题，并继续打磨海蓝暖光风格。
- 统计页改为摘要卡、生活指标、分类占比和高频商家结构。
- 设置页自动分类规则折叠为摘要卡，降低首屏密度。
- 待确认、账本、统计页加入 Material 3 下拉刷新。
- 待确认页空状态加入小票视觉锚点和上传方法入口。
- 编辑页消费时间改为日期选择器，不向普通用户显示 ISO 8601。
- 编辑页把标签、评分、后悔指数和 OCR 原文折叠到更多记录。
- 删除待确认账单前必须二次确认。
- 统计页增加最近 7 天趋势柱状图，基于本地 confirmed 缓存计算。
- 账本页增加当前筛选摘要和空状态小票图形。
- 设置页将清缓存、清绑定归入风险操作，并补充清绑定二次确认。
- 编辑页消费时间支持分别选择日期和时分，继续保持 UTC 保存、本地显示。
- 统计页总览卡增加月环比，基于本地 confirmed 缓存计算，不增加图表依赖。
- 统计页增加本地月预算进度；预算只存在本机设置，统计页展示剩余/超支和进度条。
- 设置页自动分类规则拆入独立管理面，主设置页只保留摘要和“管理”入口。

## P1 下一批

- 统计页后续可补充更细的分类洞察，但隐藏 0 数据。
- 设置页首屏继续检查是否还有可合并或可折叠的维护信息。

## P2 后续

- 建立统一字体、颜色、间距和圆角规范。
- 增加页面切换动画和列表手势。
- 分类体系扩展为餐饮、交通、购物、娱乐、医疗、教育、住房、通讯、其他，并支持自定义。

## 不回退决定

- 金额仍然使用 `amount_cents` / `amountCents`，不使用浮点金额。
- Android 首版仍然以 iPhone 快捷指令上传为主，不把“拍照导入”做成假入口。
- 技术诊断可以保留，但默认隐藏在“查看详情”后面。
