# 0024 三端 UI/UX 美化统一

日期：2026-05-12

## 背景

`v0.4-alpha2` 已经把小票夹拆成三端体验：

- Android：生活流，围绕上传、待确认、账本和统计。
- `/web`：桌面账本流，围绕 Review Center、Transaction Center、Reports 和批量处理。
- `/owner`：本机管理流，围绕服务状态、设备、上传链接、备份和诊断。

三端定位不同，但如果视觉语言、状态词、卡片节奏和空状态长期分裂，用户会感觉像三个系统。后续 v0.8/v0.9 加入预算、报表、图表和 Dashboard 后，这个问题会更明显。

## 决策

三端 UI/UX 采用"统一设计语言，不强行同屏复刻"：

- Android 保持生活化、触摸优先、适合单手操作。
- `/web` 保持桌面效率、批量处理、密度更高。
- `/owner` 保持本机运维定位，但视觉上使用同一套小票夹状态、色彩语义和布局节奏，不再像零散脚本页。

统一项：

- 信息结构：交易卡字段顺序保持"缩略图 -> 商家 -> 金额 -> 分类 -> 时间 -> 来源 -> 状态 -> 账本"。
- 状态词：待确认、缺金额、缺商家、疑似重复、已确认、已忽略、离线、备份过期、设备正常、设备已停用。
- 颜色语义：金额、超支、提醒、成功、风险、禁用、离线、重复提示在三端含义一致。
- 组件节奏：卡片圆角、间距、分组标题、状态 badge、主次按钮层级保持同一设计家族。
- 空状态 / 加载态 / 错误态：文案生活化，不暴露 endpoint、token、Cloudflare、端口、脚本名或本机路径。
- 图表视觉：预算、趋势、排行、目标进度的图例、颜色语义、长标签截断和空数据提示保持一致。

## 版本落点

- `v0.8`：建立预算、统计卡片和基础 Dashboard 的三端视觉基线。
- `v0.9`：完成报表、图表、Goals 和 Dashboard 卡片的三端 UI/UX polish。
- `v1.0`：做三端 UI 一致性回归，形成发布级视觉检查清单和截图基线。

## 约束

- 不为了统一视觉引入跨端大前端框架。
- 不把 `/owner` 改成普通用户入口；它仍然只允许 loopback。
- 不在普通用户 UI 暴露服务器地址、token、UploadLink、Cloudflare、端口、接口名、脚本名、日志路径或本机真实路径。
- 不用前端样式掩盖权限、账本隔离、分页、统计口径或错误处理缺陷。
- 不为了视觉统一破坏 Android MVVM、后端 route/service 边界或 `/web` loopback 边界。

## 验收

三端 UI/UX 统一任务完成时必须至少检查：

- Android、`/web`、`/owner` 的首页 / 列表 / 详情 / 空状态 / 错误态截图。
- 深色模式对比度、长中文商家名、长账本名、金额极值和无数据月份。
- 待确认、已确认、疑似重复、离线、备份过期等状态在三端的文案和颜色语义。
- Android 窄屏、桌面常规宽度和 `/owner` 管理表格的布局稳定性。
- 公网边界：`/web` 和 `/owner` 远程仍 403。

## 后果

后续 UI polish 不再只看单端好不好看，而要同时看三端是否像同一个产品。实现仍允许三端布局不同，但状态语义、视觉节奏和用户可读文案必须一致。
