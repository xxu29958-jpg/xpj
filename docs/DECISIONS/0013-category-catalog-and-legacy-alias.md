# 0013 分类目录与旧分类兼容

日期：2026-05-05

## 背景

首版分类里使用了 `吃饭`，同时也保留了 `AI订阅`、`数码`、`游戏` 等更贴合小票夹使用场景的分类。随着账本页、统计页和自动分类规则变多，如果默认分类和旧分类长期并存，会造成筛选、统计和导出口径分裂。

## 决策

默认分类统一为：

```text
餐饮
交通
购物
娱乐
医疗
教育
住房
通讯
AI订阅
数码
游戏
生活
其他
```

`吃饭` 作为旧版兼容别名归一到 `餐饮`。

后端负责：

- 新建、修改、自动分类规则命中时归一分类。
- 分类列表返回标准默认分类和已有自定义分类。
- 统计和筛选使用归一后的分类。
- 启动时把已有 `Expense` 和 `CategoryRule` 中的旧分类迁移为新分类。

Android 负责：

- 本地提供同一套默认分类，避免网络暂时不可用时分类选择空白。
- 服务端返回旧分类时在 DTO/Entity/Domain 转换层归一。
- UI 保存草稿时归一分类。

## 结果

账本、统计、导出、自动分类和 Android 本地缓存使用同一个分类口径。历史数据仍可兼容，但普通界面不再展示 `吃饭` 这个旧分类。
