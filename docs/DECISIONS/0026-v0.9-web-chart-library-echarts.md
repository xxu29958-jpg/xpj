# 0026 v0.9 /web 图表库 ECharts

日期：2026-05-14

## 背景

v0.9 的 `/web/reports` 已经接入真实 `reports_overview` 服务，展示趋势、商家排行和分类环比。继续用手写柱状条可以保留降级展示，但难以覆盖 tooltip、动态缩放、PNG 预览和后续更复杂的报表交互。

`0023-chart-library-policy.md` 允许 v0.9 在完成依赖审计后引入展示层图表库。本决策只覆盖本机 `/web` 展示层，不改变后端报表 API、统计口径、权限或账本隔离。

## 决策

`/web/reports` 引入 Apache ECharts 6.0.0：

```text
backend/app/static/web/vendor/echarts.min.js
backend/app/static/web/vendor/echarts.LICENSE
```

使用方式：

- ECharts 文件从 npm tarball 解包后自托管，禁止 CDN。
- `reports.js` 只读取模板注入的 `report` JSON，并在浏览器端渲染趋势、商家排行和分类环比。
- 后端仍返回普通结构化数据，不返回 ECharts 私有 option，不生成图表图片。
- 原有 server-rendered 趋势条、商家列表和分类对比保留为无 JS / 图表加载失败时的回退。

## 依赖审计

一手来源：

- Apache ECharts 官网：`https://echarts.apache.org/`
- npm 元数据：`https://registry.npmjs.org/echarts/6.0.0`
- npm tarball：`https://registry.npmjs.org/echarts/-/echarts-6.0.0.tgz`

审计结论：

- `6.0.0` npm 元数据 license 为 `Apache-2.0`。
- vendor 文件来自 tarball 中的 `package/dist/echarts.min.js`，许可证来自 `package/LICENSE`。
- 文件只进入 `/static/web/vendor/`，不引入 Node 构建链，不影响 Android、后端 service、数据库模型或 API 契约。
- `/web` 仍由 loopback gate 控制；静态图表库不包含用户数据、token、本机路径或上传图片。

## 约束

- 不合并 Claude 草稿中的 sample report service、fake repository 或平行 DTO。
- 不使用 CDN，不在运行期联网拉脚本。
- 不因为图表加载失败阻塞上传、确认入账、预算、Goals、家庭权限或 CSV 导出。
- 图表颜色读取共享 design tokens；长商家名和分类名在图表中截断，完整数据仍在 server-rendered fallback 中保留。
