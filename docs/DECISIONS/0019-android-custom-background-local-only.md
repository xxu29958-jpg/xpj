# 0019 Android 自定义背景只做本机个性化

日期：2026-05-06

## 决策

小票夹 Android 支持“自定义背景 + 沉浸模式”，但它只属于本机 UI 个性化层，不进入服务端数据模型。

实现方式：

- 使用 Android Photo Picker 选择单张图片。
- Picker 返回的 `Uri` 只用于读取。
- 选中后复制到 `context.filesDir/backgrounds/custom_background.jpg`。
- 使用 DataStore 保存背景来源、私有文件路径、沉浸强度、视差开关和减少动效开关。
- 使用统一 `ImmersiveBackgroundScaffold` 包裹 App Shell。
- 按页面角色限制背景可见度：待确认/统计更沉浸，账本中等，编辑/设置/认证更克制。

## 原因

背景负责情绪，卡片负责信息，按钮负责效率。自定义背景不能破坏私人账本的核心可读性，也不能把用户相册图片扩散到服务器、导出文件或数据库。

## 约束

- 不上传背景图片。
- 不保存外部图库 `Uri` 作为长期引用。
- 不申请不必要的相册权限。
- 不让正文、金额、主按钮、错误/成功/警告颜色跟随背景随机变化。
- 不在每个 Screen 单独实现背景。
- 大图解码必须采样，避免相册原图造成内存压力。

## 后续

可增加内置背景图库、背景裁剪、预览页和轻量主色提取。但主色提取只能影响点缀，不得改变金额、主按钮、错误/成功/警告等语义色。
