# 0012 App 错误文案与诊断日志分离

## 决策

Android App 主流程只显示生活化短文案，不直接暴露工程诊断细节。

网络失败的具体原因写入 Android Logcat，tag 为 `TicketboxNetwork`。

## 原因

小票夹是私人生活 App，不是后台管理系统。绑定页、列表页和弹窗里出现 DNS、TLS、Cloudflare Tunnel、localhost、接口名等词，会让用户觉得像开发工具。

但调试时仍然需要定位问题，所以技术信息必须保留在日志或连接检测流程中。

## 规则

- 主流程文案简短、中文、用户可理解。
- 普通界面用“绑定账本”“绑定码”“连接状态”等生活化措辞，不直接显示 `Token`、`Bearer`、session token、UploadLink 等工程词。
- 普通界面用“识别原文”和“外观预览”，不直接显示 `OCR`、`JSON`、`UUID` 等工程词。
- 普通网络失败提示只说明“暂时连不上小票夹”，不把 DNS、TLS、Cloudflare Tunnel 或接口名显示给用户。
- 兜底错误文案要给出下一步，不用“操作失败”“加载失败”这类系统式短句。
- 技术原因进入 Logcat，不进入普通提示。
- 设置页诊断默认展示概览，必要时再看日志。
- 后端日志只记录已经到达后端的请求；手机连不上域名时，后端不会收到请求，因此 Android 本地日志仍然必要。

## 后果

用户看到的是 App 化文案，开发和联调仍能通过日志判断是地址、DNS、HTTPS、超时还是 Tunnel 问题。
