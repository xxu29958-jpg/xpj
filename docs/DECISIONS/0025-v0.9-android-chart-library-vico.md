# 0025 v0.9 Android 图表库 Vico

> Superseded by: [0055 Android 图表回收到原生 Canvas 与设计 token](0055-android-chart-native-canvas-tokens.md)

日期：2026-05-14

## 背景

v0.9 的报表页面开始展示服务端聚合的趋势、商家排行、分类环比和 Goals。继续在 Android 端长期手写所有图表，会让移动端交互、轴标签、深色模式和后续导出预览的维护成本变高。

已有 `0023-chart-library-policy.md` 允许 v0.9 在完成依赖审计和 ADR 后引入展示层图表库。本次只落 Android 端 Vico；`/web` 如果后续接入 ECharts 或其他图表库，需要独立完成离线 vendor、许可证、包体积和 loopback 边界审计。

## 决策

Android 端引入 Vico 3.1.0：

```text
com.patrykandpatrick.vico:compose-m3:3.1.0
```

使用方式：

- 依赖只进入 `android/gradle/libs.versions.toml` 和 `android/app/build.gradle.kts`。
- 图表只在 Compose UI 展示层使用，当前落点是统计页 v0.9 报表卡片。
- 数据仍来自后端 `/api/reports/overview` 的结构化响应，经 `ReportsRepository` 和 domain model 进入 `StatsViewModel`。
- 后端不返回 Vico 私有模型，不渲染图表图片。
- 原有本地最近 7 天趋势和统计卡保留，作为报表接口失败时的降级展示。

## 依赖审计

一手来源：

- 官方文档：`https://github.com/patrykandpatrick/vico/tree/master/guide/v3.x.x`
- Maven metadata：`https://repo1.maven.org/maven2/com/patrykandpatrick/vico/compose-m3/maven-metadata.xml`
- 3.1.0 POM：`https://repo1.maven.org/maven2/com/patrykandpatrick/vico/compose-m3/3.1.0/compose-m3-3.1.0.pom`

审计结论：

- `3.1.0` 是当前 metadata 中最新稳定版本；`3.2.0-next.4` 属于 next 预发布线，不进入主线。
- POM license 为 Apache License 2.0。
- POM 项目地址为 Vico 官方 GitHub 仓库。
- 依赖只用于 Android 展示层，不进入后端、Repository 统计口径、权限、账本隔离或数据库模型。

## 约束

- 不复制 Claude 草稿里的 fake repository、平行 domain model 或手写 Canvas 趋势图。
- 不为了 Vico 改变 `/api/reports/overview` 契约。
- 不因为图表加载失败阻塞上传、待确认、确认入账、账本同步、预算或家庭权限。
- 图表轴标签、空状态、长商家名和深色模式要使用现有 UI token 与中文文案。
